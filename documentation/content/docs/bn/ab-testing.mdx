## এটা কী করে

A/B টেস্টিং দিয়ে একই কন্টাক্ট গ্রুপের বিভিন্ন অংশে দুটি ভিন্ন মেসেজ ভেরিয়েন্ট পাঠিয়ে কোনটি ভালো পারফর্ম করে তা পরিমাপ করা যায়। চি-স্কোয়ার্ড সিগনিফিকেন্স টেস্ট ব্যবহার করা হয়।

## ধারণা

| টার্ম | অর্থ |
|-------|-------|
| ভেরিয়েন্ট A | প্রথম মেসেজ |
| ভেরিয়েন্ট B | দ্বিতীয় মেসেজ (চ্যালেঞ্জার) |
| স্প্লিট রেশিও | A পাবে এমন কন্টাক্টের অনুপাত (ডিফল্ট `0.5`) |
| সিগনিফিকেন্স | p-value < 0.05 মানে পার্থক্য র‍্যান্ডম নয় |

## ধাপসমূহ

### ১. ক্যাম্পেইন তৈরি

```json
POST /api/v1/campaigns
{
  "name": "ঈদ অফার — A/B টেস্ট",
  "sessionId": "uuid",
  "groupId": "uuid"
}
```

### ২. A/B টেস্ট সেটআপ

```json
POST /api/v1/campaigns/:id/ab-test
{
  "variantA": "uuid-বাংলা-টেমপ্লেট",
  "variantB": "uuid-ইংরেজি-টেমপ্লেট",
  "splitRatio": 0.5
}
```

### ৩. ক্যাম্পেইন শুরু ও রেজাল্ট পর্যবেক্ষণ

```
GET /api/v1/campaigns/:id/ab-test/results
```

### সিগনিফিকেন্স পড়ুন

| মেসেজ | অর্থ |
|-------|-------|
| "পর্যাপ্ত ডেটা নেই" | প্রতি ভেরিয়েন্টে < ১০০ ডেলিভার |
| "তাৎপর্যপূর্ণ পার্থক্য নেই" | একই রকম পারফরম্যান্স |
| "ভেরিয়েন্ট A ভালো" | ভবিষ্যতে A ব্যবহার করুন |
| "ভেরিয়েন্ট B ভালো" | ভবিষ্যতে B ব্যবহার করুন |

## API এন্ডপয়েন্ট

```
POST   /api/v1/campaigns/:id/ab-test            — A/B টেস্ট সেটআপ
GET    /api/v1/campaigns/:id/ab-test            — টেস্ট কনফিগ
GET    /api/v1/campaigns/:id/ab-test/results    — লাইভ রেজাল্ট
PATCH  /api/v1/campaigns/:id/ab-test            — স্ট্যাটাস আপডেট
```
