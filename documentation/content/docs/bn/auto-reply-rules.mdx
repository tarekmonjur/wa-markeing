## এটা কী করে

অটো-রিপ্লাই রুল কিওয়ার্ডের ভিত্তিতে ইনবাউন্ড হোয়াটসঅ্যাপ মেসেজে স্বয়ংক্রিয়ভাবে রিপ্লাই পাঠায়। সিস্টেম অপ্ট-আউট (STOP) এবং অপ্ট-ইন (START) কিওয়ার্ডও স্বয়ংক্রিয়ভাবে হ্যান্ডেল করে।

## ম্যাচ টাইপ

| টাইপ | আচরণ | উদাহরণ |
|------|-------|---------|
| `EXACT` | সম্পূর্ণ মেসেজ কিওয়ার্ডের সমান | `stop` → `"STOP"` ম্যাচ, `"please stop"` না |
| `CONTAINS` | মেসেজের যেকোনো জায়গায় কিওয়ার্ড | `price` → `"দাম কত?"` |
| `STARTS_WITH` | মেসেজ কিওয়ার্ড দিয়ে শুরু | `hi` → `"Hi there"` |
| `REGEX` | ECMAScript রেগেক্স প্যাটার্ন | `order\s*\d+` → `"order 12345"` |

সব ম্যাচিং **কেস-ইনসেনসিটিভ**।

## সিস্টেম অপ্ট-আউট কিওয়ার্ড

এই কিওয়ার্ডগুলো স্বয়ংক্রিয়ভাবে কন্টাক্ট অপ্ট-আউট করে:

```
stop · unsubscribe · cancel · end · quit · বন্ধ
```

## সিস্টেম অপ্ট-ইন কিওয়ার্ড

```
start · yes · শুরু
```

## রুল তৈরি করুন

```json
POST /api/v1/auto-reply-rules
{
  "keyword": "দাম",
  "matchType": "CONTAINS",
  "replyBody": "আমাদের মূল্য তালিকার জন্য এখানে ক্লিক করুন: https://example.com/price",
  "priority": 10,
  "isActive": true
}
```

### নির্দিষ্ট সেশনে সীমাবদ্ধ (ঐচ্ছিক)

```json
{
  "sessionId": "uuid",
  "keyword": "অর্ডার",
  "matchType": "CONTAINS",
  "replyBody": "আপনার অর্ডার নম্বরটি জানান।"
}
```

## REGEX নিরাপত্তা

ইউজার-সাপ্লাইড রেগেক্স প্যাটার্ন **ReDoS** আক্রমণ প্রতিরোধে যাচাই করা হয়।

## API এন্ডপয়েন্ট

```
POST   /api/v1/auto-reply-rules          — রুল তৈরি
GET    /api/v1/auto-reply-rules          — সব রুল দেখুন
PATCH  /api/v1/auto-reply-rules/:id      — রুল আপডেট
DELETE /api/v1/auto-reply-rules/:id      — রুল মুছুন
```

## সমস্যা সমাধান

| সমস্যা | কারণ | সমাধান |
|--------|------|--------|
| রুল কাজ করছে না | `isActive: false` বা সেশন বিচ্ছিন্ন | রুল ও সেশন স্ট্যাটাস পরীক্ষা করুন |
| ভুল রুল কাজ করছে | প্রায়োরিটি সমস্যা | সঠিক রুলের প্রায়োরিটি বাড়ান |
