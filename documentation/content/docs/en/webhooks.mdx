## What This Does

Receive real-time HTTP POST notifications at a URL you control when platform events occur. Use webhooks to integrate with CRMs, Zapier, n8n, Make, or your own system.

Each delivery is signed with **HMAC-SHA256** for verification.

## Supported Events

| Event | Fired when |
|-------|-----------|
| `campaign.started` | Campaign transitions to `RUNNING` |
| `campaign.completed` | All messages processed |
| `campaign.paused` | Campaign paused mid-run |
| `campaign.failed` | Campaign cancelled or fatal error |
| `message.sent` | Message accepted by WhatsApp |
| `message.delivered` | Message delivered to device |
| `message.read` | Message read by recipient |
| `message.failed` | Message delivery failed |
| `contact.opted_out` | Contact sent STOP |

## Step-by-Step

### 1. Register an Endpoint

```json
POST /api/v1/webhooks
{
  "url": "https://your-server.com/webhook",
  "secret": "my-signing-secret-32chars-min",
  "events": ["campaign.completed", "message.failed", "contact.opted_out"]
}
```

- `url` must be **HTTPS**
- `secret` is your HMAC signing key
- `events` ‚Äî subscribe to only what you need

### 2. Verify HMAC Signature

Every delivery includes `X-Webhook-Signature`. Verify on your server:

```javascript
const crypto = require('crypto');

function verifySignature(rawBody, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('hex');
  return `sha256=${expected}` === signature;
}
```

### 3. Payload Format

```json
{
  "event": "campaign.completed",
  "timestamp": "2026-05-01T10:30:00.000Z",
  "data": {
    "campaignId": "uuid",
    "name": "Eid Offer 2026",
    "sentCount": 48,
    "deliveredCount": 45,
    "failedCount": 3
  }
}
```

## Retry Schedule

| Attempt | Delay |
|---------|-------|
| 1st | Immediate |
| 2nd | 1 minute |
| 3rd | 5 minutes |
| 4th | 30 minutes |
| 5th | 2 hours |
| After 5 | `ABANDONED` |

## SSRF Protection

- Must be `https://`
- DNS resolved IP must be public
- Private IP ranges are blocked (`10.x`, `172.16-31.x`, `192.168.x`, `127.x`)

## API Endpoints

```
POST   /api/v1/webhooks                        ‚Äî Register endpoint
GET    /api/v1/webhooks                        ‚Äî List endpoints
GET    /api/v1/webhooks/:id                    ‚Äî Get endpoint
PATCH  /api/v1/webhooks/:id                    ‚Äî Update endpoint
DELETE /api/v1/webhooks/:id                    ‚Äî Delete endpoint
GET    /api/v1/webhooks/:id/deliveries         ‚Äî Delivery history
POST   /api/v1/webhooks/:id/test               ‚Äî Send test ping
```

## Limitations

- Maximum 10 endpoints per user
- Delivery timeout: 10 seconds per attempt
- `secret` is write-only (cannot be read back)
