## What This Does

Attach images, videos, audio clips, and documents to message templates. When a campaign runs, the media file is sent alongside the template text via WhatsApp.

## Supported File Types

| Category | MIME Types | Max Size |
|----------|-----------|----------|
| **IMAGE** | `image/jpeg`, `image/png`, `image/webp`, `image/gif` | 16 MB |
| **VIDEO** | `video/mp4`, `video/3gpp` | 100 MB |
| **AUDIO** | `audio/aac`, `audio/mp4`, `audio/mpeg`, `audio/amr`, `audio/ogg` | 16 MB |
| **DOCUMENT** | `application/pdf`, `.doc/.docx`, `.xls/.xlsx` | 100 MB |

## Upload Flow

```
1. POST /api/v1/media/upload  (multipart/form-data)
2. Server validates MIME type via magic bytes
3. File streamed to MinIO (never buffered in memory)
4. Response: { mediaId, url, type, mimeType, size }
5. Use mediaId when creating templates
```

## Step-by-Step

### 1. Upload a File

```bash
curl -X POST http://localhost:3001/api/v1/media/upload \
  -H "Authorization: Bearer <token>" \
  -F "file=@./eid-offer.jpg"
```

### 2. Attach Media to a Template

```json
POST /api/v1/templates
{
  "name": "Eid Offer with Banner",
  "body": "আস্সালামু আলাইকুম {{name}}! ঈদ উপলক্ষে ২০% ছাড়।",
  "mediaUrl": "https://minio.../wa-media/eid-offer.jpg",
  "mediaType": "IMAGE"
}
```

### 3. Use in a Campaign

Campaigns automatically pick up media from the template. Each contact receives the image + caption with their variables substituted.

## Security — Extension Spoofing Protection

The server validates actual file content using **magic bytes**, not the file extension. A file renamed from `malware.exe` to `promo.jpg` will be rejected.

| Format | Magic bytes |
|--------|-------------|
| JPEG | `0xFF 0xD8 0xFF` |
| PNG | `0x89 0x50 0x4E 0x47` |
| PDF | `0x25 0x50 0x44 0x46` |
| MP4 | `ftyp` box at byte 4 |

## API Endpoints

```
POST /api/v1/media/upload      — Upload file → presigned URL
GET  /api/v1/media/:mediaId    — Get fresh presigned URL
```

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| `415 Unsupported Media Type` | File type not allowed | Check supported types |
| `413 Payload Too Large` | File exceeds size limit | Compress or resize |
| Presigned URL expired | URL older than 1 hour | Call `GET /media/:mediaId` |
