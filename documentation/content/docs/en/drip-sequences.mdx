## What This Does

A drip sequence is a series of time-delayed messages sent to contacts automatically. Define the steps (each with a template, delay, and condition), enroll contacts, and the system handles the rest.

**Use cases:** onboarding new customers, follow-up nudge sequences, Ramadan sehri/iftar promotions.

## Concepts

### Step Conditions

| Condition | Behaviour |
|-----------|-----------|
| `ALWAYS` | Send regardless of contact activity |
| `NO_REPLY` | Skip if contact sent any inbound message since enrollment |
| `REPLIED` | Only send if the contact has replied since enrollment |

### Enrollment Statuses

| Status | Meaning |
|--------|---------|
| `ACTIVE` | Receiving steps normally |
| `PAUSED` | Steps paused (admin action) |
| `COMPLETED` | All steps sent |
| `UNSUBSCRIBED` | Contact sent STOP |

## Step-by-Step

### 1. Create a Sequence

```json
POST /api/v1/drip-sequences
{
  "name": "New Customer Onboarding",
  "isActive": true
}
```

### 2. Add Steps

```json
POST /api/v1/drip-sequences/:id/steps
{ "stepNumber": 1, "templateId": "uuid-welcome", "delayHours": 0, "condition": "ALWAYS" }

POST /api/v1/drip-sequences/:id/steps
{ "stepNumber": 2, "templateId": "uuid-discount", "delayHours": 24, "condition": "NO_REPLY" }

POST /api/v1/drip-sequences/:id/steps
{ "stepNumber": 3, "templateId": "uuid-last-chance", "delayHours": 72, "condition": "NO_REPLY" }
```

**Sequence flow:**

```
[Step 1: Welcome] → +24h → [Step 2: Discount (skip if replied)]
                                    │
                                   +72h
                                    │
                          [Step 3: Last Chance (skip if replied)]
```

### 3. Enroll Contacts

```json
POST /api/v1/drip-sequences/:id/enroll
{
  "contactIds": ["uuid1", "uuid2", "uuid3"],
  "sessionId": "wa-session-uuid"
}
```

- Opted-out contacts are silently skipped
- Re-enrolling is idempotent
- Step 1 fires immediately

### Example: 3-Day Ramadan Sequence

| Step | Template | Delay | Condition |
|------|----------|-------|-----------|
| 1 | Sehri menu message | 0h | ALWAYS |
| 2 | Iftar special offer | 12h | ALWAYS |
| 3 | Full-day meal package | 24h | NO_REPLY |

## API Endpoints

```
POST   /api/v1/drip-sequences                  — Create sequence
GET    /api/v1/drip-sequences                  — List sequences
GET    /api/v1/drip-sequences/:id              — Get sequence + stats
PATCH  /api/v1/drip-sequences/:id              — Update sequence
DELETE /api/v1/drip-sequences/:id              — Delete sequence

POST   /api/v1/drip-sequences/:id/steps        — Add a step
PATCH  /api/v1/drip-sequences/:id/steps/:stepId — Update a step
DELETE /api/v1/drip-sequences/:id/steps/:stepId — Remove a step

POST   /api/v1/drip-sequences/:id/enroll       — Enroll contacts
GET    /api/v1/drip-sequences/:id/enrollments  — List enrollments
```

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| Contact not enrolled | Contact is opted-out | Check `contact.optedOut` |
| Step 2 never fires | `NO_REPLY` and contact replied | Expected behaviour |
| Enrollment shows `UNSUBSCRIBED` | Contact sent STOP | Expected behaviour |
